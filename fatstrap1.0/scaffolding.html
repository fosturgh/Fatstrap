lazystrap Global settings

Requires HTML5 doctype

lazystrap makes use of certain HTML elements and CSS properties that require the use of the HTML5 doctype. Include it at the beginning of all your projects.

<!DOCTYPE html>
<html lang="en">
  ...
</html>

Typography and links

Like Bootstrap, lazystrap sets basic global display, typography, and link styles. Specifically, we:

  • Continue to Remove margin on the body
  • Set background-color: white; on the body
  • Use the @baseFontFamily, @baseFontSize, and @baseLineHeight attributes as our typographic base
  • Set the global link color via @linkColor and apply link underlines only on :hover

These styles can be found within scaffolding.less.

Reset via Normalize

Normalize Normalize.css a project by Nicolas Gallagher and Jonathan Neal that also powers the HTML5 Boilerplate.

While Bootstrap use much of Normalize within their reset.less they have removed some elements specifically for Bootstrap, we agreed with this in principal so followed suit, but just incase included a copy of the full normalize.css kit and kaboodle in our download for you to add at if you need to.

Infinite Gridding

The grid allows for nesting down as far as you'd like, though at a certain point it will get absurd. You can use this nesting to create quite complex layouts, as well as some other tricks like form layouts or visual elements.

          	
          	<div class="row">
  <div class="active-8 columns">8
    <div class="row">
      <div class="active-8 columns">8 Nested
        <div class="row">
          <div class="active-8 columns">8 Nested Again</div>
          <div class="active-4 columns">4</div>
        </div>
      </div>
      <div class="active-4 columns">4</div>
    </div>
  </div>
  <div class="active-4 columns">4</div>
</div>

Offsetting Portions of the Grid

Offsets allow you to create additional space between columns in a row. Like the rest of the grid, they're nestable. You can use classes like .lazy-offset-1 and .active-offset-3 to manipulate your column positions in different ways. You can offset up to 11 since you wouldn't ever offset a full-width column.

          	
		<div class="row">
  <div class="lazy-1 columns">1</div>
  <div class="lazy-11 columns">11</div>
</div>
<div class="row">
  <div class="lazy-1 columns">1</div>
  <div class="lazy-10 lazy-offset-1 columns">10, offset 1</div>
</div>
<div class="row">
  <div class="lazy-1 columns">1</div>
  <div class="lazy-9 lazy-offset-2 columns">9, offset 2</div>
</div>

  <div class="lazy-1 columns">1</div>
  <div class="lazy-8 lazy-offset-3 columns">8, offset 3</div>
</div>

Offsetting Portions of the Grid

Centered columns are placed in the middle of the row. This does not center their content, but centers the grid element, itself. This is accomplished by adding a class of lazy-centered or active-centered depending on which breakpoint you want the columns to center on. active versions will carry through all breakpoints if not overridden by a lazy version. You can center any number of columns you have. To have a column not carry it's active-centered to lazy, add a class of lazy-uncentered.

          	
          	<div class="row">
  <div class="active-3 active-centered columns">3 centered</div>
</div>
<div class="row">
  <div class="active-6 lazy-centered columns">6 centered</div>
</div>
<div class="row">
  <div class="active-9 active-centered lazy-uncentered columns">9 centered</div>
</div>
<div class="row">
  <div class="active-11 active-centered columns">11 centered</div>
</div>
          </div>

'Sauce' Ordering

Using these source ordering classes, you can shift columns around between our breakpoints. This means you can have a sub-nav on the left on large displays, but go below on small ones. .push-# is the syntax you'll use.

  <div class="active-3 active-centered columns">3 centered</div>
</div>
<div class="row">
  <div class="active-6 lazy-centered columns">6 centered</div>

Small Grid

As you've probably noticed in the examples above, you have access to a active and lazy grid. If you know that your grid structure will be the same for active devices as it will be on lazy devices, just use the active grid. You can override your active grid classes by adding lazy grid classes to it.

          	
          	<div class="row">
  <div class="active-2 columns">2</div>
  <div class="active-10 columns">10, last</div>
</div>
<div class="row">
  <div class="active-3 columns">3</div>
  <div class="active-9 columns">9, last</div>
</div>
